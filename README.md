# pdf_parse_for_RAG_text_new

將 PDF 文件轉換成適合 RAG（Retrieval-Augmented Generation）使用的文字格式。本專案的核心概念是以「截圖 / 版面保留」的思路處理 PDF 內容，降低傳統 PDF 文字抽取時常見的斷行錯亂、表格跑版、段落順序錯誤等問題，讓後續切 chunk、建立向量索引與問答檢索更穩定。

## 專案目標

PDF 文件常包含多欄排版、表格、頁首頁尾、圖片註解或特殊格式。若直接使用一般文字抽取工具，輸出的內容可能不適合直接放入 RAG 流程。

本專案希望將 PDF 轉換成更容易被 LLM 與向量資料庫使用的文字資料，讓文件能更順利進入以下流程：

1. PDF 解析
2. 文字清理
3. 段落整理
4. Chunk 切分
5. Embedding 建立
6. RAG 檢索與問答

## 功能特色

- 將 PDF 內容整理成 RAG 友善的文字格式
- 以接近「截圖理解版面」的方式保留文件結構
- 適合用於知識庫建置、文件問答、企業內部規章查詢等場景
- 可作為向量資料庫前處理流程的一部分
- 方便後續銜接 自託管Dify 

## 適用場景

- PDF 文件轉文字
- RAG 知識庫前處理
- 法規、規章、SOP、合約等長文件解析
- 需要保留段落語意與版面順序的文件處理
- 將 PDF 內容轉成可切 chunk、可 embedding 的乾淨文字
